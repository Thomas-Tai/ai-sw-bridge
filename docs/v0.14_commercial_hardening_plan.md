# v0.14 Commercial Hardening Plan

**Status:** DRAFT — pre-execution. Authored 2026-05-28.
**Branch baseline:** `feat/v0.13-sonnet-glm-batch` at `40cf26f`.
**Test baseline:** 947 passing on Python 3.10/3.12/3.14 (per `README.md:210`).
**Intent:** One campaign that lifts the bridge from B+ engineering to commercial-deployable. No follow-up "v0.15 cleanup" expected — the leftovers move into `docs/DEFERRED.md`, not into the next sprint.

---

## 1. Why this plan exists

The 2026-05-28 audit (this document's parent) surfaced four shipped correctness bugs, seven points of README drift, two flat modules that should be classes, and one 0.1.0-era function the codebase still wires up despite its own docstring admitting it doesn't work. None of these are show-stoppers individually; collectively they are the difference between "interesting alpha" and "product a paying SOLIDWORKS shop can deploy with a straight face."

This plan executes all of them in one branch, behind one version bump (`0.13.0 → 0.14.0`), with one CHANGELOG entry. Commercial users get one breaking change to absorb instead of three.

## 2. Scope and non-scope

**In scope (this campaign):**
- §A correctness bugs (4 items)
- §B README/USAGE/AGENTS doc parity (7 items)
- §C class refactor: `observe.py` → `SolidWorksObserver`; `mutate.py` → `Proposal` + `ProposalStore`
- §E1: removal of `sw_run_macro` (breaking)
- §F commercial-readiness: env-var matrix, stability-tier review, migration guide, semver bump
- Verification: full test suite + import-linter + manual MMP build in all three modes

**Explicitly deferred to v0.15+ (logged in `docs/DEFERRED.md`):**
- `_build_*` handler classification (mirror, pattern, revolve, hole, fillet, chamfer). The `FEATURE_REGISTRY` dict already provides polymorphism; class ceremony adds no value here.
- `sw_types.py` (762-line constants module) decomposition.
- `SW_PREF_INPUT_DIM_VAL_ON_CREATE` cleanup — author marked it "harmless, may help on other SW builds."
- Switching from late-binding `pywin32` to `comtypes`/typelib (deferred since v0.10).

#### C1 — `SolidWorksObserver`

**New file:** `src/ai_sw_bridge/observe.py` becomes the class-based form; existing module stays at the same path (lazy-loaded by `__init__.py`).

**Design:**
```python
class SolidWorksObserver:
    """Read-only observation of the active SOLIDWORKS document.

    All methods return JSON-shaped dicts with a consistent envelope:
    ``{"ok": bool, "error": str | None, ...method-specific keys...}``.

    The class is stateless — instances exist only to scope the
    SW-Application provider, which is overridable for tests.
    """

    def __init__(
        self,
        *,
        app_provider: Callable[[], Any] = get_sw_app,
        doc_resolver: Callable[[Any], Any | None] = get_active_doc,
    ) -> None:
        self._app = app_provider
        self._doc = doc_resolver

    # --- template -----------------------------------------------------

    def _with_active_doc(
        self,
        fn: Callable[[Any], dict[str, Any]],
        *,
        empty_template: dict[str, Any],
        requires_part: bool = False,
        requires_assembly: bool = False,
    ) -> dict[str, Any]:
        """Get app→doc→run fn(doc) under the shared error boundary."""
        result = dict(empty_template)
        try:
            sw = self._app()
            doc = self._doc(sw)
            if doc is None:
                result["error"] = "no_active_doc"
                return result
            if requires_part and not self._is_part(doc):
                result["error"] = f"requires_part (active doc is {self._doc_type_name(doc)})"
                return result
            if requires_assembly and not self._is_assembly(doc):
                result["error"] = f"requires_assembly (active doc is {self._doc_type_name(doc)})"
                return result
            method_payload = fn(doc)
            result.update(method_payload)
            if result.get("error") is None:
                result["ok"] = True
            return result
        except Exception as exc:
            result["error"] = f"dispatch failed: {exc!r}"
            return result

    # --- public API ---------------------------------------------------

    def active_doc(self) -> dict[str, Any]: ...
    def feature_errors(self) -> dict[str, Any]: ...
    def equations(self) -> dict[str, Any]: ...
    def bbox(self) -> dict[str, Any]: ...                    # requires_part=True
    def volume(self) -> dict[str, Any]: ...                  # requires_part=True
    def screenshot(self, width=640, height=360, fit_view=False, filename=None) -> dict[str, Any]: ...
    def mate_errors(self) -> dict[str, Any]: ...             # requires_assembly=True
    def custom_props(self) -> dict[str, Any]: ...
    def measure(self, entity_a=None, entity_b=None) -> dict[str, Any]: ...
    def enabled_addins(self) -> dict[str, Any]: ...
```

**Hard rename — caller updates:**
- `cli/observe.py`: each subcommand handler becomes `SolidWorksObserver().METHOD(...)`.
- `mcp/_tool_observe.py`: each `@com_tool` wraps `SolidWorksObserver().METHOD(...)`.
- `mcp/_tool_build.py` and `cli/build.py`: `sw_get_enabled_addins()` → `SolidWorksObserver().enabled_addins()`.
- `__init__.py`: `_LAZY_MODULES` stays; the lazy module still exports a module object — but consumers should import the class.
- Tests: bulk rewrite — `from ai_sw_bridge.observe import sw_get_X` → `from ai_sw_bridge.observe import SolidWorksObserver` plus call-site update.

**Migration tip in CHANGELOG:**

#### C2 — `Proposal` + `ProposalStore`

**Design:**
```python
class ProposalState(str, Enum):
    PROPOSED = "proposed"
    DRY_RUN_OK = "dry_run_ok"
    DRY_RUN_BROKE = "dry_run_broke"
    COMMITTED = "committed"
    UNDONE = "undone"


@dataclass
class Proposal:
    proposal_id: str
    created_at: float
    doc_path: str | None
    locals_path: Path
    var: str
    old_expression: str
    new_expression: str
    line_index: int
    snapshot_text: str
    state: ProposalState = ProposalState.PROPOSED
    dry_run_result: dict[str, Any] | None = None
    committed_at: float | None = None
    undone_at: float | None = None

    def to_json(self) -> dict[str, Any]: ...
    @classmethod
    def from_json(cls, d: dict[str, Any]) -> "Proposal": ...


class ProposalStore:
    """File-backed proposal lifecycle store.

    One JSON per proposal under ``root/`` (default: ./proposals, override
    via ``AI_SW_BRIDGE_PROPOSALS``). Acquires ExclusiveLock around every
    read/write to keep the *_locals.txt single-source-of-truth invariant.
    """

    def __init__(
        self,
        *,
        root: Path | None = None,
        app_provider: Callable[[], Any] = get_sw_app,
        doc_resolver: Callable[[Any], Any | None] = get_active_doc,
    ) -> None: ...

    def propose(self, var: str, new_value: str) -> dict[str, Any]: ...
    def dry_run(self, proposal_id: str) -> dict[str, Any]: ...
    def commit(self, proposal_id: str) -> dict[str, Any]: ...
    def undo_last(self) -> dict[str, Any]: ...

    # Internal: shared force-rebuild + manager-status read; no longer
    # top-level module functions.
    def _force_rebuild(self, doc) -> tuple[bool, str | None]: ...
    def _read_manager_status(self, doc) -> int | None: ...
    def _read_var_value(self, doc, var_name: str) -> float | str | None: ...
    def _get_linked_locals(self, doc) -> Path | None: ...
```

- `cli/mutate.py`: every `sw_propose_local_change(...)` → `ProposalStore().propose(...)`, etc.
